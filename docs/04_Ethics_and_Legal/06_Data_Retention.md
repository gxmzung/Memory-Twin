# 06. Data Retention Policy

## 데이터 보관 원칙

Memory Twin은 사용자가 설정한 보관 기간과 동의 범위에 따라 데이터를 보관한다.

## 보관 기간 예시

| 데이터 | 기본 보관 기간 | 삭제 요청 시 |
|---|---|---|
| 계정 정보 | 서비스 이용 기간 | 30일 내 삭제 |
| 사진/영상 | 사용자 설정 기간 | 즉시 비활성 후 삭제 |
| 음성 데이터 | 사용자 설정 기간 | 즉시 비활성 후 삭제 |
| AI 대화 로그 | 1년 또는 사용자 설정 | 삭제 가능 |
| 접근 로그 | 보안 목적 보관 | 법정 기간 후 삭제 |
| Legacy Letter | 예약 공개 전까지 | 작성자 삭제 가능 |

## 삭제 절차

1. 사용자가 삭제 요청
2. 데이터 즉시 비활성화
3. 복구 대기 기간 30일
4. 원본 데이터 삭제
5. 백업 데이터 순차 삭제
6. 삭제 완료 로그 기록

## 주의사항

- 사용자가 명시적으로 영구 삭제를 요청하면 복구 불가 안내 후 삭제한다.
- 사후에도 생전 삭제 요청 설정이 우선한다.
- AI 임베딩 및 벡터 데이터도 함께 삭제해야 한다.
