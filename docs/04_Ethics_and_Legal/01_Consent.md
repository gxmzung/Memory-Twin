# 01. Consent Policy

## 목적

Memory Twin은 생전 동의를 기반으로 데이터를 수집·저장·처리·공개한다.  
동의는 전체 동의가 아니라 항목별 선택 동의로 관리한다.

## 동의 항목

| 항목 | 설명 | 필수 여부 |
|---|---|---|
| 사진 저장 동의 | 사진 업로드 및 보관 | 선택 |
| 음성 저장 동의 | 음성 파일 저장 및 분석 | 선택 |
| 영상 저장 동의 | 영상 업로드 및 보관 | 선택 |
| 문서 저장 동의 | 일기, 편지, 메모 저장 | 선택 |
| AI 검색 동의 | 업로드 데이터를 AI 검색에 활용 | 선택 |
| AI 대화 동의 | 기록 기반 대화형 응답 생성 | 선택 |
| 가족 공개 동의 | 지정 가족에게 데이터 공개 | 선택 |
| 사후 활성화 동의 | 사망 이후 지정 조건에서 서비스 활성화 | 선택 |
| Legacy Letter 공개 동의 | 예약 편지 공개 | 선택 |

## 동의 원칙

- 동의는 언제든 철회 가능해야 한다.
- 동의 철회 시 해당 기능은 즉시 비활성화한다.
- 생전 동의가 없는 데이터는 AI 대화에 사용하지 않는다.
- 유족 요청보다 생전 본인 동의가 우선한다.

## 구현 요구사항

- Consent 테이블을 별도로 둔다.
- 동의 변경 이력을 저장한다.
- 동의 상태는 API 요청마다 확인한다.
- 동의 없는 데이터는 RAG 검색 대상에서 제외한다.
