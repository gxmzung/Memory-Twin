Purpose

The purpose of this knowledge base is to continuously improve as an engineer.

I want to track:

* feedback from real engineers
* repeated weaknesses
* technical study priorities
* career decisions
* company and industry research
* mentor advice
* lessons from interviews and code reviews

Core Principles

1. Understand Before Writing

Do not memorize code.

Understand why it works.

вё»

2. Explain Every Design Decision

Every project should answer:

* Why did I build this?
* What problem does it solve?
* What is the architecture?
* How does data flow?
* What did I implement?
* What do I not understand yet?
* What would I improve next?

вё»

3. Build Depth

Do not only create more repositories.

Deepen the projects I already have.

вё»

4. Learn From Real Engineers

Every interview, coffee chat, and code review is feedback data.

Record it.

Reflect on it.

Improve.
